# Security policy

## Supported versions

MicDeck is currently in public preview. Security fixes are provided for the
latest release only.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's **Security → Advisories → Report a vulnerability** flow so the
report and follow-up remain private. Include:

- affected MicDeck version;
- Windows version and architecture;
- reproduction steps or a minimal proof of concept;
- expected impact;
- any suggested mitigation.

You should receive an acknowledgement within 72 hours. Please allow time for a
fix and coordinated disclosure before publishing details.

## Scope

Reports about MicDeck's Rust, C++, JavaScript, IPC, installer orchestration, or
update/download behavior are in scope. Vulnerabilities in unmodified
third-party software should also be reported to that component's vendor.
