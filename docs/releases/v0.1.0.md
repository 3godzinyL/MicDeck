# MicDeck v0.1.0 — Public Preview

MicDeck's first public preview turns a Windows PC into a focused voice-chat
audio desk: sound pads, desktop audio, and a physical microphone leave through
one virtual input.

## Highlights

- New MicDeck interface with Library, Live Studio, and Settings workspaces
- Soundboard playback for MP3, WAV, FLAC, OGG, AAC, and M4A
- Quick Capture for YouTube, YouTube Shorts, and TikTok audio
- Persistent per-sound global hotkeys with an in-app shortcut recorder
- Background imports and waveform analysis that keep the interface responsive
- One-click WASAPI loopback for Spotify, browsers, games, and desktop audio
- Native Rust + C++20 audio pipeline with adaptive `IAudioClient3` periods
- Live signal meters, estimated latency, underrun count, and engine status
- Guided VB-CABLE setup and managed `MicDeck Virtual Mic` endpoint
- Full English and Polish interfaces, close-to-tray, autostart, and optional cursor glow
- Portable and per-user NSIS builds

## Download

- `MicDeck-Setup.exe` — recommended per-user installer
- `MicDeck-portable.exe` — portable application build
- `SHA256SUMS.txt` — checksums for both binaries

## Before you start

1. MicDeck currently supports Windows 10/11 x64.
2. Builds are not code-signed yet, so Windows SmartScreen may show a warning.
3. The VB-CABLE driver may require administrator approval and a Windows restart.
4. URL capture requires `yt-dlp` and FFmpeg in `PATH`.

## Known limitations

- Per-application capture is not included in this preview; system-audio sharing currently uses the full Windows render mix.
- Voice-processing options in Discord and similar apps may suppress music or
  effects; lower or disable those filters when sharing non-speech audio.

Please report reproducible bugs through the included bug form. Report suspected
security vulnerabilities privately through GitHub Security Advisories.
